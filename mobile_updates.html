Recommended mobile analytics workflow

Use the workbench like a lightweight implementation design review. The goal is not just a code snippet; it is agreement on what the app should measure, where the data lands, and how it proves business value.

1 Name the mobile moment

Define the screen or action in user language, then choose trackAction, trackState, or Edge sendEvent.

2 Define the value

State why the event matters: conversion, adoption, engagement, retention, or troubleshooting.

3 Map the data

Connect each mobile field to XDM and the legacy Analytics variable it replaces or supports.

4 Generate the code

Give iOS and Android teams a concrete payload with identity and datastream assumptions visible.

5 Validate the contract

Confirm app behavior, Edge forwarding, Analytics values, CJA readiness, and downstream activation use.
