What this tool does

During a Web SDK migration, teams often know what the legacy tag used to send, but not whether the same measurement intent exists in the new implementation. This comparator checks whether each legacy eVar, prop, event, tag, or data layer value has a target Web SDK/XDM destination.

1
Paste legacy mappingTealium iQ tags, AppMeasurement variables, SDR rows, or simple CSV/key-value lines.
2
Paste target mappingWeb SDK, XDM, data.__adobe.analytics, or CJA-ready target paths.
3
Review parityMapped, missing, needs-detail, and new mappings become a backlog the team can execute.

How to read the results

Mapped
The legacy variable exists in the target and has a documented XDM or data path.
Needs detail
The target variable exists, but the implementation path is unclear. Engineering needs a destination path before build.
Missing
A legacy variable has no target mapping. Decide whether to migrate, retire, or replace it.
New
A Web SDK/XDM mapping exists that did not exist in legacy. Confirm it is intentional and governed.
